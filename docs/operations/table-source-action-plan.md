## Classification Rules

- `official_scraped`: must be populated from official source only. Accepted
  sources are AEAT, BOE/BORME, EUR-Lex, CNMV, Banco de Espana, SEPBLAC, AEPD,
  DGT, ESMA/EBA/EIOPA/EFRAG/EPC/IRS/OECD/ICAC where applicable.
- `derived_internal`: generated from already-ingested official rows. It must
  keep lineage to the source record and, when possible, the sync job.
- `operational_internal`: generated by system activity: audit logs, eval runs,
  alerts, human review, incidents, customer/entity workflows. Empty can be
  healthy when no event has happened.
- `configured_but_unavailable`: official source exists, but ingestion is not
  implemented, not configured, blocked by upstream access, or intentionally
  out of current scope. The MCP must say unavailable, not infer.
- API/MCP exposure rule: every empty table surfaced to clients must include
  `availability_status`, `reason`, `row_count`, `registry_classification`,
  `target_path`, `action`, and `safe_to_answer`; no empty response may be used
  as evidence that no obligation, registry entry, deadline, rate, or filing
  exists.

## Ralph Stories

1. `TS-001`: make the MCP classification-aware.
   - Add/verify a table-domain registry that marks `official_scraped`,
     `derived_internal`, `operational_internal`, or
     `configured_but_unavailable`.
   - Acceptance: every public table is classified; unclassified count is zero.

2. `TS-002`: close local-to-VPS drift for P0 official/derived tables.
   - Run the P0 workers on VPS or deploy missing code first.
   - Status: completed for P0 on 2026-05-10.
   - Acceptance: each P0 VPS table has non-zero rows or a documented official
     upstream blocker; no fake seeds.

3. `TS-003`: expose availability metadata to MCP/API.
   - Tool responses must know whether a domain is populated, operational-empty,
     or unavailable.
   - Acceptance: unavailable domains return explicit uncertainty rather than
     an empty/hallucinated answer.

4. `TS-004`: enforce freshness and lineage gates.
   - `source_freshness_snapshot` and `data_lineage` must be generated after
     ingestions.
   - Acceptance: stale official-source domains produce metadata warnings.

5. `TS-005`: keep GitHub and VPS aligned.
   - Commit/push plan and scripts.
   - Copy deployed docs/artifacts to `/srv/esdata` or pull from GitHub.
   - Acceptance: local `git rev-parse HEAD`, VPS deployed revision, and GitHub
     branch agree, or drift is documented.
