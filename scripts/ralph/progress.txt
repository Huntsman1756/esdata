=== Iteration A-01 — 2026-05-09 22:45 ===
Story: A-01 Fix Decimal serialization bug in /v1/legislacion/buscar (P0 bloqueante)
Branch: fix/audit-critical
Verification run: curl /v1/legislacion/buscar?q=IVA (200 + resultados LIVA), curl /v1/consulta?q=IVA (200), curl /v1/legislacion/LIVA/articulos/1 (200 + boe_reference) = regression check
Result: PASS
Fix applied:
- apps/api/routers/buscar.py: replaced `return JSONResponse(content=result)` with `return result` in both buscar_legislacion and buscar_legislacion_hybrid. FastAPI's default jsonable_encoder handles Decimal; JSONResponse uses stdlib json.dumps which does not.
- apps/api/routers/consulta.py line 1148: removed redundant `logger = logging.getLogger(__name__)` reassignment. The module-level logger at line 24 was being shadowed because Python treats any name assigned anywhere in a function as local in the entire function → UnboundLocalError when line 1010 used it.
Learnings:
- **Pattern:** Never wrap SQLAlchemy result dicts in `JSONResponse(content=...)` when they contain numeric types. Use `return result` and let FastAPI's jsonable_encoder handle Decimal→float.
- **Gotcha:** Python function scope — reassigning a module-level name inside a function (even inside try/except) makes it local for the whole function. If you need a logger variant, name it differently.
- **Test:** /v1/consulta was a separate bug in the same audit finding; "Decimal serialization" was only half the story — the traceback mentioned line 161 of buscar.py; but /v1/consulta threw UnboundLocalError. Both now fixed.

Story: A-05 Routers sin datos devuelven status=not_available explícito (P0 — no falsos vacíos)
Branch: fix/audit-critical
Verification run: batch curl 10 endpoints (/v1/mica/casp, /v1/dora/ict-risk-registers, /v1/mifid/best-execution-records, /v1/psd2/aisp, /v1/mar/insider-transactions, /v1/priips/kids, /v1/csrd/entity-reports, /v1/sfdr/products, /v1/pbc/obligated-subjects, /v1/fraud/incidents) — 10/10 devuelven {"status":"not_available","domain":...,"message":...} con HTTP 200. Regression check /v1/legislacion/LIVA/articulos/1 sigue devolviendo datos con boe_reference.
Result: PASS 10/10 dominios cubiertos + 0 regression
Fix applied:
- apps/api/middleware/domain_availability.py NEW: DomainAvailabilityMiddleware interceptor con registry DOMAIN_PATH_TABLE_MAP (47 entradas) cubriendo MiCA, DORA, MiFID, PSD2, MAR, PRIIPs, CSRD, SFDR, PBC/AML, fraud. Cache TTL 60s para evitar latencia. Sólo intercepta GETs de paths exactos (startswith). Detail endpoints /{id} no interceptan (deben devolver 404 natural).
- apps/api/services/domain_availability.py NEW: helper check_domain() disponible para usos inline si futuros endpoints necesitan el check explícito (no usado por el middleware pero útil para casos edge).
- apps/api/main.py: registro de DomainAvailabilityMiddleware + expose engine en app.state.engine para el middleware.
Learnings:
- **Pattern:** Middleware >>> 29 modificaciones de router. Un registry centralizado es mantenible; modificaciones per-endpoint son frágiles y propensas a olvido cuando se añade un nuevo framework regulatorio.
- **Pattern:** Cache TTL corto (60s) amortiza latencia sin arriesgar datos stale — crons son weekly/daily, no real-time.
- **Pattern:** JSONResponse con status 200 bypass response_model. Usar 503 sería semánticamente más correcto (service unavailable) pero rompería clientes que interpretan cualquier no-2xx como error. Preferir 200 con status:not_available en body.
- **Gotcha:** response_model de FastAPI coerce el dict y elimina campos extra — JSONResponse directo lo evita. Middleware evita este problema por naturaleza.
- **Legal impact:** Fix crítico. Antes, un abogado que buscaba regulación de criptoactivos en MiCA recibía `[]` → "no hay regulación" → asesoramiento incorrecto. Ahora recibe "MiCA: fuente pendiente de ingesta" → derivación a fuente oficial.
